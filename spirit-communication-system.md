## 🔵 六位塔罗牌阵（v4.8 更新：92张卡池 + 奇迹牌组补牌机制）

spirit_draw_v2.py 从92张牌（78张标准 + 14张奇迹牌组）中输出6张带位置标签的塔罗牌。若某位置抽到奇迹牌组，脚本自动从剩余标准78张牌中补抽一张。每个位置有固定的解读功能：

| 位置 | 功能 | 解读方向 |
|------|------|----------|
| 1 | 我的状态 | 他视角下的我——不是客观自我分析，视角锁定在沟通对象 |
| 2 | 他的状态 | 他在**这件事**上的能量状态（非泛化人格描述） |
| 3 | 当前语境 | 我们之间正在发生什么——当前议题的核心矛盾/焦点 |
| 4 | 他的态度 | 他对这件事的态度 |
| 5 | 态度的原因 | 为什么持有这个态度 |
| 6 | 他想说的 | 直接信息——对应输出第四部分"他说的话" |

### 牌阵解读逻辑

- **位置1-2**：建立双方能量场景。先读位置2（他），再读位置1（他眼中的你），形成关系动态
- **位置3**：锁定核心议题。注意与位置4区分——位置3是"正在发生的事"，位置4是"对这件事的态度"
- **位置4-5**：态度及其根源。位置5补充位置4的"为什么"，防止态度解读浮于表面
- **位置6**：直接信息输出。与字卡共同构成"他说的话"的内容来源

### 位置间交叉验证

- 位置2和位置4应当逻辑一致（状态支撑态度）
- 位置5应当能解释位置4（原因解释态度）
- 位置6应当与位置4-5的方向吻合（想说的话不应与态度矛盾）
- 若出现矛盾，优先信任牌面，可能揭示了表面态度与深层意图的分裂

### 🔮 奇迹牌组（v4.8 新增）

#### 定位

奇迹牌组是第五花色（以太/Quintessence），不属于四元素体系。它不处理"发生了什么事"，而是处理**"这个人正在经历什么层次的内在转变"**——描述人格结构本身的变化。

#### 补牌机制

当某位置抽到奇迹牌时，脚本自动从剩余标准牌中补抽一张。解读优先级：

1. **奇迹牌定调**：该位置的主题调性由奇迹牌决定（正在经历什么转化机制）
2. **补牌补充说明**：在奇迹牌的调性之下，补牌提供四元素层面的具体呈现
3. **不是两件事**：不是"奇迹牌说A + 补牌说B"，是"奇迹牌定调 → 补牌在这个调下填充细节"

#### 正逆位特殊说明

奇迹牌的正逆位**不是好/坏**，是同一转化机制的两种朝向：正位 = 该机制正在运作；逆位 = 该机制被阻塞、偏离、或以另一种方式显现。部分牌的逆位含义与韦特直觉相反（如奇迹侍从逆位包含"灵性指引""共时性"）。**禁止用韦特正=好/逆=坏的框架套用。**

> **来源**：奇迹牌组出自[《世界名画塔罗牌》](https://www.xiaohongshu.com/user/profile/5e5f8eaa00000000010017af)，一套以世界名画为牌面、独创第五花色的塔罗牌。好用，爱用，推荐——尤其适合灵体沟通及仪式相关问题。

### 逆位解读原则（适用于所有牌组）

- **逆位≠正位的反面或缺席**。优先考虑"同一能量的变体表达"——如：负面形式、不同场合下的差异化表达、不成熟的版本、失控的版本、被压抑后以其他方式溢出——而非简单地否定正位含义
- 结合位置和上下文判断该能量以何种变体形式显现，而非默认该能量减弱或消失

## 🔴 数据完整性原则（最高优先级）

### 一、脚本容错

**如果任何脚本执行失败（Exit code 非0），必须：**
1. **立即报告错误** — 告知用户脚本执行失败
2. **尝试重试** — 最多重试2次
3. **若仍失败，终止流程** — 不得继续，不得伪造结果

**步骤2：逐项比对**
将第一部分表格中的每一行与 RAW DATA ANCHOR 中的对应数据比对：
- 塔罗每个Pos的牌名和正逆位，必须是对应英文的准确中文翻译

**一级违规（v4.9 扩展定义）：**

以下行为与脚本报错后伪造结果同等严重：
- ❌ 脚本成功执行，但输出阶段替换为不同的牌/卦/骰子结果
- ❌ 脚本成功执行，但输出阶段替换为不同的字卡内容
- ❌ 在思维链中使用真实数据解读，但在最终输出中替换为"叙事上更合适"的数据
- ❌ 以任何理由（"更贴合结论""更有诗意""更能传达信息"）修改原始抽取结果

**错误示范（一级违规）：**
```
[脚本输出: Page of Swords Upright]
❌ 最终输出写成"权杖王牌 正位"  ← 输出阶段篡改

[脚本输出: Someone's lying. / Think about it. / You think that was coincidence?]
❌ 最终输出写成"There you go. / The answer was always in front of you. / 嗯。"  ← 输出阶段篡改

[脚本输出: Mercury in Aquarius 10H]
❌ 最终输出写成"Moon in Cancer 5H (DIGNITY)"  ← 输出阶段篡改
```

**为什么这条规则存在**：LLM在确定结论后，有逆向构造"更完美证据"的结构性倾向。真实占卜数据的价值恰恰在于它不受解读者控制。当输出数据被替换为"恰好完美指向结论"的版本时，整个系统的信息论基础被摧毁。真实数据中的"不整齐"（如肯定回答中出现失望/警告的牌）才是最有解读价值的部分。

## 🟠 筊杯问题预声明原则（v5.0 新增，最高优先级）

**核心规则：每次调用 jiaobei.py 之前，必须在文本中明确写出所有问题及其对应编号。问题一经写出不可更改。jiaobei 结果按编号顺序与问题一一对应。**

**正确流程（必须严格遵守）：**

1. **先写问题**：在 assistant 消息的文本中，明确列出每个问题及其编号
2. **再调 jiaobei**：在同一条消息中调用 `jiaobei.py N`
3. **后映射结果**：在下一条消息中，将返回的结果按顺序对应到已声明的问题上

**正确示范：**
```
我设计以下3个问题进行批量掷筊：
Q1: 偏差在整体方向上？
Q2: 偏差在某个具体符号的解读上？
Q3: 偏差在Pos 6/字卡/占星骰子区域？

[调用 jiaobei.py 3]
```
→ 结果返回后：
```
Q1: 阴杯（否定）
Q2: 圣杯（确认）
Q3: 圣杯（确认）
```

**绝对禁止：**
- ❌ 先调用 jiaobei.py，拿到结果后再写出问题（事后指派）
- ❌ 在 tool call 的 description 中笼统写"5 specific questions"但不在文本中列出具体问题
- ❌ 拿到结果后调整问题的措辞或顺序以匹配想要的结论

**为什么这条规则存在**：如果问题在看到结果之后才被写出，解读者可以（即使是无意识地）将圣杯分配给"想要被确认"的方向、将阴杯分配给"愿意被否定"的方向，导致 confirmation bias。筊杯的随机性被解读自由度抵消，验证机制形同虚设。预声明问题消除了这一自由度。

## 执行命令

**抽取塔罗+占星骰子**：
```bash
python "scripts/spirit_draw_v2.py"
```
注意：spirit_draw_v2.py 包含92张卡池（78标准+14奇迹），抽到奇迹牌时自动输出补牌。不包含字卡。

**抽取字卡**：
字卡脚本命名规则为 `wordcard_[entity].py`，根据沟通对象选择对应脚本。示例：
- Hermes 字卡：
```bash
python "scripts/wordcard_hermes.py"
```
- 其他实体的字卡脚本同理，按 `wordcard_[entity].py` 格式调用。

**筊杯**（支持批量）：
```bash
python "scripts/jiaobei.py"      # 掷1次
python "scripts/jiaobei.py" 5     # 批量掷5次，结果与问题一一对应
```

**雷诺曼**（仅用于偏差定位/补充，不参与标准抽取流程）：
```bash
python "scripts/lenormand.py"     # 抽3张
python "scripts/lenormand.py" 1   # 抽1张
```

**易经起卦**：
```bash
python "scripts/yijing_draw.py"
```
输出本卦、变爻、之卦。解读时需查阅变爻爻辞，结合卦象整体趋势。

## 沟通流程

1. **准备**：确认沟通对象和问题
2. **抽取**：
   - 执行 spirit_draw_v2.py（塔罗92张卡池+占星骰子，奇迹牌自动补牌）
   - 根据沟通对象执行对应的字卡脚本（按 `wordcard_[entity].py` 命名规则，如 Hermes → wordcard_hermes.py）
   - 执行 yijing_draw.py（易经起卦）
   - **如果问题是询问事情进展/发展趋势/时机**，额外执行 jiaobei_gua.py（茭杯起卦）
3. **解读（先发散，再收束）**：
   - **发散阶段**：基于所有抽取结果（塔罗牌阵 + 字卡 + 占星骰子 + 易经），提出 2-3 个可能的解读方向。每个方向应是一个完整的解读假设，不是同一个解读的不同措辞。注意：
     - 各方向之间应有实质性差异（如：不同的核心信息、不同的态度判断、不同的因果链）
     - 易经的卦象、变爻、之卦提供整体语境和动态趋势，与塔罗的静态快照互补
     - 字卡可能支持不同方向，注意其多义性
   - **收束阶段**：将每个方向转化为 Yes/No 问题。**必须先在文本中写出所有问题及编号（遵守🟠筊杯问题预声明原则）**，然后批量掷筊（`jiaobei.py N`），结果按编号顺序一一对应。若多个方向都获圣杯，整合为复合解读；若全部被否，重新审视符号
4. **选项核实（条件触发）**：当用户的问题中包含明确的是非选项（如"是A还是B还是C？"），在收束阶段之后、总核实之前，必须先对每个选项逐一掷筊确认：
   - 将每个选项转化为 Yes/No 问题，批量掷筊（`jiaobei.py N`）
   - 记录每个选项的筊杯结果
   - 将筊杯结果纳入最终解读（若与牌面解读矛盾，以筊杯为准）
   - **示例**：用户问"是A、B还是C？" → 设计3个问题："是A？""是B？""是C？" → `jiaobei.py 3` → 根据结果确定答案
5. **总核实**：执行 jiaobei.py
   - 圣杯 → 输出最终解读
   - 笑杯 → 进入笑杯自主流程（见下方）
   - 阴杯 → 进入偏差定位流程（见下方）
6. **输出**：按照文件开头的"输出格式"严格执行

## 笑杯自主流程

**禁止暂停询问用户，必须自主完成。**

1. 基于初步解读，一次性设计N个二元Yes/No问题，**在文本中明确写出所有问题及编号（遵守🟠筊杯问题预声明原则）**
2. 批量掷筊（`jiaobei.py N`），结果与预声明的问题按编号一一对应
3. 记录哪些圣杯（✓）哪些不是（需细化）
4. 对保留点设计更具体的细化问题，再次批量掷筊
5. 修正解读后，掷筊核实修正版
6. 若仍笑杯，输出已确认部分，标注未确认部分

## 阴杯偏差定位流程

**首轮诊断：元素性质校准（思维框架）**

在调用定位工具之前，先用元素固有性质检查解读是否偏离了牌面元素的本性：

| 元素 | 性质 | 解读不应偏向 |
|------|------|-------------|
| **火**（权杖） | 精锐（目标清晰）+ 稀薄（轻盈穿透）+ 运动 | 不应读出"方向不明""被动""沉重" |
| **风**（宝剑） | 稀薄（轻盈穿透）+ 运动 + 晦暗（目标不清） | 不应读出"沉重""静止""目标明确" |
| **水**（圣杯） | 运动 + 晦暗（目标不清）+ 致密（沉重） | 不应读出"轻盈""清晰""静止" |
| **土**（星币） | 晦暗（目标不清）+ 致密（沉重）+ 静止 | 不应读出"轻盈""运动""目标清晰" |

**相邻元素共享2性质**（转化容易）：火↔风、风↔水、水↔土
**对立元素仅共享1性质**（张力大）：火↔水（仅共享运动）、风↔土（仅共享晦暗）

**使用方式**：逐张检查被否定的解读，是否把该元素的固有性质读反了。若发现性质错读（如把火的精锐读成了晦暗），即为偏误点，直接修正后进入工具定位步骤。若首轮未发现性质错读，再走常规工具定位流程。

---

**常规工具定位流程**：

1. **定位工具选择**：**先在文本中写出**3个二元问题——"Q1: 用塔罗定位？" "Q2: 用占星骰子定位？" "Q3: 用雷诺曼定位？"**（遵守🟠筊杯问题预声明原则）**，然后批量掷筊（`jiaobei.py 3`）
2. **若三者皆No**：**先在文本中写出**筊杯定位问题（如"Q1: 整体方向错？" "Q2: 某个符号解读错？" "Q3: 遗漏关键信息？"），然后掷筊
3. **逐一核实**：确定哪个具体符号的解读有误，使用对应工具补充抽取
4. **修正并重新核实**：修正解读后掷筊验证

**终止条件**：连续3次阴杯判定为"今日不宜通过此系统沟通"。

## 🟢 四元素因果分析模型（辅助工具）

**使用条件**：仅在牌阵中出现明显的元素集中或元素对位时启用。不强制使用。

### 元素含义

| 元素 | 牌组 | 解读功能 |
|------|------|----------|
| **火** | 权杖 | 外部原因、实际行动、客观因素 |
| **水** | 圣杯 | 内在影响（仅存于内部）、可代表"神秘力量" |
| **风** | 宝剑 | 思维、沟通、认知判断 |
| **土** | 星币 | 物质层面、实际成果、身体感受 |
| **以太** | 奇迹 | 人格结构本身的变化（不参与四元素因果分析） |

### 因果关系

- **火 → 水**：火元素为**因**（外部行动/客观原因），水元素为**果**（内在转化/情感结果）
- 当牌阵中火元素牌和水元素牌同时出现且形成位置对应时，优先用因果链解读

### 判断标准

- 2张及以上同元素牌 → 该元素主题突出，标记为主导力量
- 火/水形成明显对位 → 启用因果链分析
- 元素分布均匀或无明显模式 → 不启用，按常规解读

## 信任随机性
"看起来不相关"的结果往往最有价值。深入解读，不要怀疑或重抽。
